Panels and call-to-action

Image cards

The height of image cards are defined by the image used. The content is aligned to the bottom of the image and has a gradient applied, for legibility. Note that the gradient will take the full width of its container, while the image will stretch to its natural width.

There are many styling options for these panels. Note that panels using background colours behind the text tend to use .h3 headings while those against images use .h2 (or natural heading level two elements). These rules are paritally to do with how much content is to fit inside the panel, however.

Rotating pills

Rotating pills are assembled from utility and grid classes, but need the ability to switch which corners are rounded, depending upon the viewport. In order to achieve this, the grid must linearise when the viewport is md or smaller.

Tile header

Tile headers are used to create the illusion that the heading is repeating in the background of the page. Part of this effect is created using the text-shadow property in CSS. Unfortunately, SVG elements do not cast such a shadow. This means that it is preferable to use ASCII charactors to create this effect.

A list of useful ASCII characters can be found on the icon page of this guide.

Note that the --tw CSS variable will need to be configured for each text string, so it matches the length of the text.

Image border

Image border use object-fit (plus a polyfil) to ensure that two images are stacked on top of each other, the first one in the markup creating a border and the second being the focus.

Note that the images will be stretched, if they are not high enough resolution.

The image-border class is designed to be applied to an element which already has a defined with and height - for example a Bootstrap column. The images themselves do not impose a height or width on the element. This means that an image-border element will always match the content it sits next to, even if that means distorting images.

image-border can take a modifying class of jumbo which makes the border around the inner image thicker. This is used on the home page and on product landing pages.

The normal variant is used on the blog landing page.

The height of the images is defined by how much text they sit alongside and any padding and margins added to that text. Image Borders are also used in conjunction with Bleed Grids.

Bleed grid

Bleed grids are a hybrid of full-bleed layouts and the standard grid. They only allow for two column layouts, but are configurable so that the full-bleed image can sit to the right or left.

Note that bleed grid mark-up must be placed outside of any container elements.

Configuring

The background colour of the bleed grid is selected using the classes which can be found on the colour guidelines page.

The height of the bleed grid is defined by the content. The image will stretch or shrink to fit within whatever space is left.

Images could be added via class or as inline styles. The alignment of each image may require unique configurations.

Flags

Flags are tooth-like protrusions which can be attached to the outside of elements using a combination of classes. Note that they may cause overlap with proceeding or following elements, so gutter classes should be used to prevent this from happening.

The element which contains the flag should have a punch or pos-rel class added to it.

The position of the flag itself is controlled by four classes, relating to compass directions:

  1. pos-outside-nw
  2. pos-outside-ne
  3. pos-outside-sw
  4. pos-outside-se

The flag class adds rounded corners at the top of the element. To switch these corners for the bottom of the element, add the modifying class flag-south.
